## Description
This Pull Request introduces the massive v2.0 overhaul of the quiz platform, executing a complete rebrand to **CLUELY.AI**, modernizing the tech stack, optimizing the AI engine, and completely refactoring the directory structure for better maintainability.

### 🎨 Rebranding & "Editorial Brutalism" UI/UX
- Completely abandoned the old, generic Bootstrap-like interface in favor of a bespoke **Editorial Brutalist** design system (`awwwards.css`).
- Integrated high-end typography (`Bodoni Moda` serif headers paired with stark `Inter` sans-serif).
- Added monochromatic interfaces with stark borders, deep shadows, and an "Electric Blue" (`#0047FF`) accent.
- Rewrote `error.html` completely to match the brutalist aesthetic with grain/noise backgrounds and sharp UI elements.

### 🧠 AI Engine Upgrades & Optimization
- Fixed deprecation warnings by updating the `google.generativeai` implementation logic.
- Switched the AI synthesis model to **`gemini-2.5-flash-lite`**, massively reducing quiz generation latency from ~15 seconds to under 5 seconds.
- Heavily compressed the system prompts to further optimize the JSON generation speed without sacrificing accuracy.

### 🏗️ Architectural Restructuring & File Organization
- Eliminated the monolithic folder structure. The project now utilizes a clean separation of concerns:
  - **`/backend/`**: Contains `app.py`, `ai_engine.py`, `db_manager.py`, and `requirements.txt`.
  - **`/frontend/`**: Houses all `templates/` and `static/` files.
  - **`/database/`**: Isolates the local SQLite files.
- Refactored `app.py` and `db_manager.py` with relative OS path resolutions to ensure the Flask app automatically maps to the new `frontend/` directory regardless of execution context.
- Migrated from MySQL to a lightweight, zero-configuration **SQLite3** (`aegis_quiz.db`) architecture for immediate plug-and-play local development.

### 🐛 Critical Bug Fixes
- **Waiting Room Sync Error:** Resolved the critical bug where students were trapped in the waiting room (`/waiting_room`) after the instructor initiated the exam. State synchronization is now reliably written to the SQLite database.
- **Javascript Duration Crash:** Fixed an unhandled template exception in `monitor.html` that crashed the instructor dashboard and prevented global deployment due to empty duration strings.
- **SQLite Serialization Exception:** Fixed a `TypeError: Object of type Row is not JSON serializable` in `quiz.html` by converting `sqlite3.Row` objects into serializable Python dictionaries prior to Jinja templating.
- **Unused Imports & IDE Warnings:** Cleaned up unused imports (e.g., `werkzeug.security`, `uuid`, etc.) that were triggering linter errors.

### 📖 Documentation
- Completely rewrote the `README.md`.
- Added **Mermaid.js Architecture Diagrams** detailing the WebSocket pipeline and the new SQLite Entity-Relationship Diagram (ERD).

Please review the changes. The platform is now significantly faster, more secure, and aesthetically state-of-the-art.
